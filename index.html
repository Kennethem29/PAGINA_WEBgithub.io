Programación Estructurada

¿Qué es la Programación Estructurada?

La programación estructurada es un paradigma que busca mejorar la claridad, calidad y tiempo de desarrollo de un programa mediante el uso de estructuras de control como secuencias, selecciones (if/else) y repeticiones (bucles).

Puntos Clave

  • Secuencias: Ejecución de instrucciones en orden.
  • Selecciones: Uso de condicionales (if/else) para tomar decisiones.
  • Repeticiones: Uso de bucles (for, while) para repetir acciones.
  • Funciones: Bloques de código reutilizables.

Ejemplos de Código en C#

Secuencias

Las secuencias son la ejecución de instrucciones en orden. En C#, esto se logra simplemente escribiendo las líneas de código una después de la otra.


// Ejemplo en C#
using System;

class Program {
    static void Main() {
        string nombre = "Juan";
        int edad = 25;

        Console.WriteLine("Nombre: " + nombre);
        Console.WriteLine("Edad: " + edad);
    }
}
Selecciones (if/else)

Las selecciones permiten tomar decisiones en el código. En C#, se utiliza la estructura if/else para evaluar condiciones.


// Ejemplo en C#
using System;

class Program {
    static void Main() {
        int edad = 18;

        if (edad >= 18) {
            Console.WriteLine("Eres mayor de edad.");
        } else {
            Console.WriteLine("Eres menor de edad.");
        }
    }
}
Repeticiones (bucles)

Los bucles permiten repetir acciones. En C#, se pueden usar bucles for, while o do-while.


// Ejemplo en C#
using System;

class Program {
    static void Main() {
        for (int i = 0; i < 5; i++) {
            Console.WriteLine("Iteración: " + i);
        }
    }
}
Funciones

Las funciones son bloques de código reutilizables. En C#, se definen usando la palabra clave static (para métodos de clase) o sin ella (para métodos de instancia).


// Ejemplo en C#
using System;

class Program {
    static int Sumar(int a, int b) {
        return a + b;
    }

    static void Main() {
        Console.WriteLine("Suma: " + Sumar(10, 20));
    }
}

Programación Orientada a Objetos (POO)

¿Qué es la Programación Orientada a Objetos?

La POO es un paradigma de programación que organiza el software en torno a "objetos", que son instancias de "clases". Estos objetos contienen datos (atributos) y comportamientos (métodos).

Puntos Clave

  • Clases y Objetos: Las clases son plantillas, y los objetos son instancias de esas plantillas.
  • Herencia: Permite crear nuevas clases a partir de otras existentes.
  • Polimorfismo: Los objetos pueden tomar múltiples formas.
  • Encapsulamiento: Ocultar los detalles internos de un objeto.
  • Abstracción: Simplificar la complejidad al modelar solo los aspectos relevantes.

Ejemplos de Código en C#

Clases y Objetos

Una clase es una plantilla que define las propiedades y comportamientos de un objeto. Un objeto es una instancia de una clase.


// Ejemplo en C#
using System;

class Persona {
    public string Nombre { get; set; }
    public int Edad { get; set; }

    public Persona(string nombre, int edad) {
        Nombre = nombre;
        Edad = edad;
    }

    public void Saludar() {
        Console.WriteLine("Hola, soy " + Nombre);
    }
}

class Program {
    static void Main() {
        Persona juan = new Persona("Juan", 25);
        juan.Saludar(); // Hola, soy Juan
    }
}
Herencia

La herencia permite crear nuevas clases a partir de otras existentes. La clase derivada hereda los atributos y métodos de la clase base.


// Ejemplo en C#
using System;

class Animal {
    public string Nombre { get; set; }

    public Animal(string nombre) {
        Nombre = nombre;
    }

    public virtual void HacerSonido() {
        Console.WriteLine("Sonido genérico");
    }
}

class Perro : Animal {
    public Perro(string nombre) : base(nombre) {}

    public override void HacerSonido() {
        Console.WriteLine("Guau guau");
    }
}

class Program {
    static void Main() {
        Perro miPerro = new Perro("Rex");
        miPerro.HacerSonido(); // Guau guau
    }
}
Polimorfismo

El polimorfismo permite que un objeto se comporte de múltiples formas. En C#, se logra mediante la sobrescritura de métodos.


// Ejemplo en C#
using System;

class Animal {
    public virtual void HacerSonido() {
        Console.WriteLine("Sonido genérico");
    }
}

class Perro : Animal {
    public override void HacerSonido() {
        Console.WriteLine("Guau guau");
    }
}

class Program {
    static void Main() {
        Animal miAnimal = new Animal();
        Animal miPerro = new Perro();

        miAnimal.HacerSonido(); // Sonido genérico
        miPerro.HacerSonido();  // Guau guau
    }
}
Encapsulamiento

El encapsulamiento oculta los detalles internos de un objeto. En C#, se logra usando modificadores de acceso como private.


// Ejemplo en C#
using System;

class CuentaBancaria {
    private double saldo;

    public void Depositar(double monto) {
        if (monto > 0) {
            saldo += monto;
        }
    }

    public double ObtenerSaldo() {
        return saldo;
    }
}

class Program {
    static void Main() {
        CuentaBancaria cuenta = new CuentaBancaria();
        cuenta.Depositar(1000);
        Console.WriteLine("Saldo: " + cuenta.ObtenerSaldo());
    }
}
Abstracción

La abstracción simplifica la complejidad al modelar solo los aspectos relevantes. En C#, se logra usando clases abstractas o interfaces.


// Ejemplo en C#
using System;

abstract class Vehiculo {
    public abstract void Mover();
}

class Coche : Vehiculo {
    public override void Mover() {
        Console.WriteLine("El coche se mueve.");
    }
}

class Program {
    static void Main() {
        Vehiculo miCoche = new Coche();
        miCoche.Mover(); // El coche se mueve.
    }
}
